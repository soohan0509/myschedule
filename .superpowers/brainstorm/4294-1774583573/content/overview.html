MySchedule 개선 계획 — 전체 범위

요청하신 항목들을 구현 난이도와 우선순위별로 정리했습니다. 진행 순서를 확인해주세요.

Phase 1 — 즉시 적용 가능 (Quick Wins)
✅ UX 개선
  • "오늘" 버튼 추가
  • 일정 중복 충돌 경고
  • 최근 삭제 임시 보관함 (Undo)
✅ 디자인 & 애니메이션
  • 토스트 알림 (저장/삭제 피드백)
  • 모달 fade-in / slide-up 애니메이션
  • 로딩 스켈레톤 UI
  • 빈 상태 (Empty State) 일러스트
  • 반 탭 활성 상태 명확화
✅ 보안 수정
  • 어드민 링크 — 비관리자에게 숨기기
  • XSS 방지 (입력값 이스케이프)
  • 비밀번호 정책 강화 (길이, 강도계)
  • API 에러 핸들링
✅ 접근성
  • aria-label (날짜/일정 스크린 리더)
  • 키보드 포커스 표시
  • 색상 대비 WCAG AA 검토
Phase 2 — 핵심 기능 추가 (Medium Complexity)
🔵 캘린더 뷰
  • 주간 뷰 (Weekly View)
  • 일간 뷰 (Daily View)
  • 월/주/일 토글 버튼
🔵 D-Day 카운트다운
  • D-Day 일정 타입 추가
  • 캘린더 상단 D-Day 배너
  • 사이드바 카운트다운 표시
🔵 NEIS API 연동
  • 학사일정 자동 캘린더 표시
  • 급식 확인 버튼 (시간표 옆)
  • 아침/점심/저녁 메뉴 모달
🔵 다크 모드
  • CSS 변수 기반 테마 전환
  • 토글 버튼 (헤더)
  • 시스템 설정 자동 감지
Phase 3 — 고급 기능 (Higher Complexity)
🟣 위젯/대시보드
  • 오늘 일정 요약
  • 이번 주 주요 일정
  • 다가오는 D-Day 목록
🟣 댓글/메모 기능
  • 일정별 댓글 달기
  • 알림 연동
🟣 출결/참석 확인
  • 참석/불참 응답 기능
  • 인원 파악 통계
🟣 어드민 고도화
  • 활동 로그 (감사 로그)
  • 공지사항 팝업 시스템
  • 어드민 대시보드
  • 일괄 작업 (체크박스 선택)
⚠️ 범위 외 (서버 인프라 필요)

브라우저 푸시 알림, Service Worker 오프라인 캐싱, 이미지 WebP 변환은 Vercel 서버리스 확장 또는 별도 인프라가 필요합니다. 나중에 별도로 다룰 수 있습니다.
